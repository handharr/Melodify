Architecture 101

The Layers

ViewController
Layout, user events, binds to ViewModel via Combine
↓
ViewModel
@MainActor · @Published state · owns loading + error
↓
UseCase
Business rules, validation, orchestration (async let / TaskGroup)
↓
Repository
Translates Param → Request · owns Domain↔Data boundary
↓
DataSource
Builds URL · calls APIClient · returns DTO
↓
APIClient
URLSession wrapper · GET / POST / PUT / PATCH / DELETE
Layer rule: Presentation → Domain ← Data. Domain depends on nothing.

Data Flow — Read

ViewModel.load()
  → UseCase.execute(policy: .fresh, param: SearchTracksParam)
    → Repository.searchTracks(policy:param:)
      → TrackSearchRequest(query:offset:limit:)  ← Param → Request
      → DataSource.searchTracks(request)
        → APIClient.get(url)               ← URLSession
          → iTunesSearchResponse (DTO)
      → TrackMapper.toDomain(dto) → Track  ← only Mapper knows both
    → [Track]
  → TrackUIModelMapper.toUIModel(track) → TrackUIModel
→ ViewModel.tracks = [TrackUIModel]     ← VC never sees Track

Data Flow — Write

ViewModel.createPlaylist(name:)
  → UseCase.execute(param: CreatePlaylistParam)
    → guard !name.isEmpty else throw PlaylistError.emptyName
    → Repository.createPlaylist(param:)
      → CreatePlaylistRequest(name:description:trackIds:)
      → DataSource.createPlaylist(request)
        → APIClient.post(url, body: request)   ← Encodable, id excluded
          → PlaylistDTO (response)
      → PlaylistMapper.toDomain(dto) → Playlist
→ ViewModel.playlist = PlaylistUIModel
Rule: No FetchPolicy on writes — always hit the network. FetchPolicy only answers "where do I read from?"

Key Types

Param<Query, Path>

struct Param<Query, Path> {
    let query: Query   // filter / body fields
    let path: Path     // URL path segments
}
typealias SearchTracksParam = Param<SearchTracksQuery, Void>
typealias UpdatePlaylistParam = Param<UpdatePlaylistQuery, UpdatePlaylistPath>

Adding a field to Query or Path never breaks call sites.

FetchPolicy

static let fresh  = FetchPolicy(force: true,  allowStale: false) // always network
static let cached = FetchPolicy(force: false, allowStale: true)  // cache-first
static let strict = FetchPolicy(force: false, allowStale: false) // cache only, fail if missing

Travels from ViewModel → UseCase → Repository. Read UseCases only.

Request Structs (Data layer)

struct TrackSearchRequest { let query: String; let offset: Int; let limit: Int }
struct UpdatePlaylistRequest: Encodable {
    let id: Int          // path param — excluded via CodingKeys
    let name: String
    enum CodingKeys: String, CodingKey { case name }
}

UIModel + UIModelMapper

enum TrackUIModelMapper {
    static func toUIModel(_ track: Track) -> TrackUIModel {
        let seconds = track.durationMs / 1000
        TrackUIModel(duration: String(format: "%d:%02d", seconds / 60, seconds % 60), ...)
    }
}

ViewController only ever sees UIModel. Domain entities never reach the ViewController.

Concurrency Patterns

2 concurrent fetches → async let

async let tracks = trackRepository.searchTracks(policy: policy, param: ...)
async let playlists = playlistRepository.fetchPlaylists(policy: policy)
return HomeData(featuredTracks: try await tracks, playlists: try await playlists)

N concurrent fetches → withThrowingTaskGroup

return try await withThrowingTaskGroup(of: HomeSection.self) { group in
    for (genre, query) in param.query.genreQueries {
        group.addTask { try await repository.searchTracks(policy: policy, param: ...) }
    }
    var sections: [HomeSection] = []
    for try await section in group { sections.append(section) }
    return sections.sorted { ... }  // order is non-deterministic — always sort
}

ViewModel state pattern

@MainActor final class TrackListViewModel {
    @Published private(set) var tracks: [TrackUIModel] = []
    @Published private(set) var isLoading = false
    @Published private(set) var errorMessage: String?

    func load() {
        isLoading = true
        Task {
            defer { isLoading = false }   // guaranteed on success AND failure
            do {
                tracks = try await useCase.execute(...)
            } catch {
                errorMessage = error.localizedDescription
            }
        }
    }
}

Trade-off Talking Points

Repository + DataSource vs Repository alone?

Repository owns the Domain↔Data boundary (Param→Request, DTO→Domain). DataSource owns HTTP details. Splitting them means you can swap clients or add caching without touching business mapping logic.

Param<Query, Path> vs plain arguments?

Adding a field doesn't break every call site. Path and Query have different semantic roles — typed struct makes intent explicit, type system enforces it.

No BaseUseCase protocol?

Read and write have different contracts — FetchPolicy vs none. A base type either lies about the contract or recreates what per-UseCase protocols already do with less ceremony.

UIModelMapper as a separate type?

Only when it has display formatting logic worth testing independently. A pure 1:1 field copy doesn't need a dedicated type — inline in ViewModel is cleaner. Consistency for its own sake is boilerplate.

@MainActor on ViewModel?

All @Published mutations must happen on the main thread. @MainActor enforces this at compile time instead of relying on manual DispatchQueue.main.async.
